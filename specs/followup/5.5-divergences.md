# Lua 5.5 official-suite divergence sweep (DISCOVER)

Method: each `/tmp/lua-refs/lua-5.5.0-tests/*.lua` run through
`LUA_RS_VERSION=5.5 target/debug/lua-rs` vs `/tmp/lua-refs/bin/lua5.5.0`, from
the test dir, under a driver that sets the preamble
(`_soft=true; _port=true; _nomsg=true; _U=false; arg=arg or {}`) and then
`dofile`s the test. Output+exit normalized for heap addresses / program name.
Every divergence below was re-reproduced with `specs/oracle/diff_one.sh 5.5`.

## Parity estimate

28 files swept (excluding the harness/heavy files `all`, `heavy`, `verybig`,
`cstack`, `memerr`, `files` which need a real fs / huge runtime).

- **Byte-identical: 9** — `bitwise`, `bwcoercion`, `pm`, `attrib`, `events`,
  `code`, `gengc`, `tracegc`, `api`, `big`, `main` (and `constructs` is
  effectively identical — its only diff is the nondeterministic
  `math.random(0,1)` GLOB1 line, noise).
- **Divergent: ~16.** Almost all trace to **two** root causes (the
  `global <const>` parser ambiguity and named-varargs `...t`); knock those two
  out and the divergent count drops sharply.

### Cross-cutting (surfaces in many tracebacks, all versions worth checking)
- **namewhat (item #6):** global C functions render as `[C]: in function 'error'`
  where 5.5 ref shows `[C]: in global 'error'`. Reproduced via
  `print(error())`. `auxlib.rs::push_func_name`.
- **arg-error function name:** `bad argument #3 (position out of bounds)` ours
  vs `bad argument #3 to 'utf8.offset' (position out of bounds)` ref — the
  `to '<fn>'` clause is dropped. Same `push_func_name` family. Confirm whether
  5.5-only or all-version before fixing.
